# UX/UI Evaluation — ColdChain IoT

Дата: 2026-03-08  
Основа: скриншоты Dashboard, Devices, Locations, Alerts, Onboard, Settings

---

## 1. Общая оценка

**Сильные стороны:**
- Чистая структура, понятная навигация
- 5-шаговый onboarding с прогрессом
- Фильтры на странице устройств
- Copy-кнопки для credentials
- Консистентный sidebar

**Области улучшения:**
- Визуальная иерархия и кликабельность
- Empty states
- Обратная связь при ошибках
- Связь между сущностями (локация → устройства)

## 2. Рекомендации по страницам

### 2.1 Dashboard

| Проблема | Рекомендация |
|----------|--------------|
| «Устройства (онлайн) 0/2» — неочевидно, что это кликабельно | Сделать карточки кликабельными (ссылка на /devices?status=online) |
| «Default Zone (2 устройств)» — теги устройств выглядят как метки, не как ссылки | Показывать serial как ссылку или добавить «Перейти к устройствам» |
| «Локация с большим холодильником» — «Нет зон» | Добавить CTA: «Создать зону» или «Добавить устройство» |

### 2.2 Устройства

| Проблема | Рекомендация |
|----------|--------------|
| Serial — единственная явная ссылка | Сделать кликабельной всю строку или название |
| Тестовые названия («Холодильник большой очень...») | Placeholder или валидация при вводе |
| Пустая колонка «Тревога» при отсутствии | Показать «—» или «Нет» для ясности |

### 2.3 Локации

| Проблема | Рекомендация |
|----------|--------------|
| Заголовок «Локации и зоны», но зоны не видны | Показывать зоны внутри локации или счётчик зон |
| Адрес «—» для всех | Подпись «Не указан» или скрыть колонку, если не используется |
| Связь локация ↔ устройства | Добавить счётчик устройств на локацию |

### 2.4 Оповещения

| Проблема | Рекомендация |
|----------|--------------|
| «Нет событий по выбранным фильтрам» | Добавить: «Система в норме» или «Все тревоги подтверждены» |
| «С (дата ISO)» | Заменить на «Дата начала» или «С даты» |
| При наличии алертов | Ссылка на устройство из алерта |

### 2.5 Onboard (Добавить устройство)

| Проблема | Рекомендация |
|----------|--------------|
| Шаг 3 — credentials | Уже хорошо: copy-кнопки, предупреждение «показывается один раз» |
| Шаг 4 — инструкции | Добавить ссылку на hardware-provisioning.md или краткую шпаргалку |
| Шаг 5 — проверка | Показывать «Ожидание первого сообщения...» с таймером/индикатором |

### 2.6 Настройки

| Проблема | Рекомендация |
|----------|--------------|
| Иконка Telegram у пользователя | Подсказка при наведении: «Telegram привязан» / «Привязать» |
| Модалка «Новый пользователь» | Уже понятно: обязательные поля, роль по умолчанию |

## 3. Путь пользователя

### Типичный сценарий: добавить датчик

1. **Дашборд** → «Добавить устройство» (понятно)
2. **Onboard** → 5 шагов (хорошо)
3. **Проверка** → переход на /devices, клик по serial (здесь был crash — исправлен)
4. **Деталь устройства** → график, калибровки, управление (после фикса — стабильно)

### Проблемный участок

Клик по устройству в списке → crash. **Исправлено:** defensive coding, error boundary, безопасная обработка readings.

## 5. Дизайн-система

- **Цвета:** primary (синий), success (зелёный), destructive (красный) — используются последовательно
- **Типографика:** заголовки, подписи, muted — читаемо
- **Компоненты:** Card, Badge, Button, Table — единообразны

Рекомендация: сохранять текущий стиль, дорабатывать микро-взаимодействия и пустые состояния.
